Phần lõi

Bảng checklist Onpage tổng hợp

Ba cột thống nhất: Hạng mục · Tên checklist · Mô tả chi tiết cách check. Mỗi bước là một thao tác cụ thể cần xác nhận.

Hạng mục Tên checklist Mô tả chi tiết cách check
01
Domain		 www & non-www
  1. Website đang chạy ở phiên bản nào (www hay non-www)?
  2. Nhập đủ 4 phiên bản (www/non-www × http/https) vào trình duyệt, kiểm tra URL cuối sau redirect bằng httpstatus.io.
  3. Yêu cầu: 3/4 phiên bản redirect về 1 phiên bản chính thức duy nhất (ưu tiên https + non-www).
http & https
  1. Website đã có chứng chỉ SSL chưa?
  2. Bản http đã được redirect 301 sang https chưa?
Hosting / Server
  1. Dùng Screaming Frog quét toàn bộ website.
  2. Nếu quét được mà không phát sinh lỗi kết nối → hosting/server hoạt động ổn định.
02
Metadata & Technical tags		 Title tag
  1. Mỗi trang có một title tag duy nhất?
  2. Kiểm tra mã nguồn hoặc dùng Screaming Frog.
  3. Không trùng lặp title giữa các trang.
Meta description
  1. Các trang quan trọng đã có meta description chưa?
  2. Dùng Screaming Frog tìm trang còn thiếu.
  3. Nội dung hấp dẫn, chứa từ khóa liên quan.
OG tags (Open Graph)
  1. Kiểm tra phần <head> có đủ: og:locale, og:type, og:title, og:description, og:url, og:image.
  2. Verify hiển thị khi chia sẻ bằng Facebook Debugger (developers.facebook.com/tools/debug).
Viewport
  1. Tìm trong mã nguồn thẻ <meta name="viewport" content="width=device-width, initial-scale=1">.
  2. Test giao diện hiển thị trên nhiều thiết bị khác nhau.
Charset
  1. Tìm trong mã nguồn thẻ <meta charset="UTF-8">.
  2. Đảm bảo dùng UTF-8 để hỗ trợ tiếng Việt có dấu.
Favicon
  1. Biểu tượng nhỏ có hiển thị trên tab trình duyệt chưa?
  2. Kiểm tra nhanh bằng extension SEOQuake.
Hreflang
  1. Tìm trong code thẻ <link rel="alternate" hreflang="..." href="...">.
  2. Kiểm tra bằng SEOQuake.
  3. Đảm bảo phiên bản đúng ngôn ngữ/khu vực được liên kết với nhau.
Canonical
  1. Các URL động (có tham số) đã canonical về URL gốc chưa?
  2. Tìm thẻ <link rel="canonical" href="...">.
  3. Mục tiêu: tránh nội dung trùng lặp.
Meta robots
  1. Trang quan trọng để index, follow.
  2. Trang không cần index để noindex, nofollow.
  3. Kiểm tra trong mã nguồn phần <head>.
03
Heading		 Cấu trúc heading
  1. Mỗi trang có một H1 duy nhất?
  2. Phân cấp hợp lý: H1 → H2 → H3 → H4, phản ánh đúng nội dung.
  3. Kiểm tra bằng Web Developer → Information → View Document Outline.
04
Hình ảnh & Alt		 Alt text cho ảnh
  1. Mọi hình ảnh có alt text mô tả ngắn gọn chưa?
  2. Dùng SEOQuake để tìm ảnh thiếu alt.
  3. Nên chứa từ khóa liên quan khi phù hợp.
05
Structured Data / Schema		 Khai báo schema
  1. Đã có schema phù hợp với loại trang chưa? Toàn trang: Organization, Searchbox, LocalBusiness; Danh mục: +BreadcrumbList; Sản phẩm/Bài viết: +Product/Article; FAQ/Video/Job: schema riêng.
  2. Verify bằng Google Rich Results Test (search.google.com/test/rich-results).
06
URL		 Cấu trúc URL & trang 404
  1. Phân tầng rõ ràng (category lớn / category nhỏ / sản phẩm); tránh đặt sản phẩm ở root.
  2. Dùng dấu gạch ngang -, không dùng underscore _.
  3. Trang 404 đáp ứng cơ bản: có link về trang chủ và gợi ý nội dung.
07
Internal link & Navigation		 Liên kết nội bộ & breadcrumb
  1. Trang quan trọng có internal link từ trang khác trỏ tới chưa?
  2. Không có liên kết hỏng (404) — kiểm tra bằng Screaming Frog.
  3. Breadcrumb đúng dạng: Home › Category lớn › Category nhỏ › Tiêu đề (mục cuối không link).
08
External link		 Liên kết ra ngoài
  1. Dùng Screaming Frog lọc external link.
  2. Không trỏ tới trang lỗi (3xx / 4xx / 5xx).
  3. Toàn bộ external link gắn thuộc tính rel="nofollow".
09
Robots.txt & Sitemap		 Robots.txt
  1. Có file robots.txt đúng định dạng Google.
  2. Bắt buộc khai báo link tới sitemap.xml trong file.
Sitemap.xml
  1. Có sitemap XML đúng định dạng, không URL trùng lặp.
  2. Cập nhật real-time khi có bài/sản phẩm mới.
  3. Đã submit lên Google Search Console & Bing Webmaster.
10
Tốc độ & Hiệu suất		 Tốc độ tải & Core Web Vitals
  1. PageSpeed Insights đạt ≥ 60/100 trên cả desktop & mobile.
  2. Core Web Vitals (FCP, LCP, TBT, CLS) đạt tiêu chuẩn.
  3. Cache tự động xóa sau khi cập nhật, không gây lỗi CSS.

Sáu hạng mục được nhấn mạnh trong onboarding: Domain, OG tags, Viewport, Charset, Favicon, Hreflang. Các hạng mục còn lại bổ sung để sổ tay đủ hệ thống cho một lượt rà soát onpage hoàn chỉnh.

Tự kiểm tra

5 câu trắc nghiệm cuối bài

Làm nhanh để ghi nhớ và xác minh bạn đã nắm bài. Chọn đáp án rồi bấm “Xem kết quả”.

Câu 1

Khi kiểm tra Domain, yêu cầu bao nhiêu phiên bản phải redirect về một phiên bản chính thức?

Đáp án: C. 3/4 phiên bản redirect về 1 phiên bản duy nhất, ưu tiên https + non-www.
Câu 2

Thẻ meta nào đảm bảo website hiển thị đúng trên thiết bị di động?

Đáp án: B. Thẻ viewport (width=device-width, initial-scale=1) điều khiển hiển thị trên thiết bị di động.
Câu 3

Charset nào cần dùng để hỗ trợ tiếng Việt có dấu?

Đáp án: A. UTF-8 hỗ trợ đầy đủ tiếng Việt có dấu.
Câu 4

Dùng công cụ nào để verify OG tags trước khi chia sẻ link lên Facebook?

Đáp án: D. Facebook Debugger (developers.facebook.com/tools/debug) hiển thị OG tags khi chia sẻ.
Câu 5

Toàn bộ external link nên gắn thuộc tính rel nào?

Đáp án: C. External link gắn rel="nofollow" theo checklist của SEONGON.
